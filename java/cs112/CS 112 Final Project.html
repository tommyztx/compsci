CS 112 Final Project

A. Goals

=20

This open-ended project provides you the o= pportunity to apply everything you have learned in CS 112 to a program of your choice. You will be responsible for choosing a problem that you can solve using a Java program, analyzing that problem to determine how to solve it, decomposing the functionality of your program to identify its class structure and the API for each class, and then implementing and testing your program. Note that your project must use in some way 1) Object Orientation, 2) Linked Lists, and 3) Recursion. Recusion can be used within your Linked List implementation to fullfil both objectives.

B. Working with a Partner

=20

You are encouraged to work with one partne= r on this project (but not more than one). Your partner must be currently enrolled in CS 112, but does not have to be in the same section. =20

Both partners are expected to contribute equally to this project. If we determine that the work was not evenly distributed between the partners, we reserve the right to allocate grades proportional to the work done by each partner. Additionally, we reserve the right to allocate different, seperate grades to either partner during grading for other reasons. =20

Only ONE partner per project should submit your work on behalf of both partners -- the partner who submits should be the person whose NetID comes first in alphabetical order. If both partners on a team submit, the team will receive a point deduction, since it causes us problems when grading. =20

The one exception to the above rule is that BOTH partners must INDEPENDENTLY submit a brief survey on their participation and their partner's participation in the project. =20

You are ONLY allowed to share code and discuss the project with your partner; you may NOT collaborate with anyone besides your partner beyond what is normally permitted. The course collaboration policy still applies. =20

You need to determine a way to share code with your partner in a private, secure, and easily accessible manner. We strongly recommend that you create a shared private DropBox or Google Drive folder for this project and use it to share code. If you are familiar with GitHub, feel free to set up a private repository for your project. Emailing files back and forth will not typically work well. Be careful of synchronization issues though with DropBox/Google Drive -- if both of you edit the same file at the same time, DropBox/Google Drive will not merge the changes together correction, so one person's work may be overwritten. The BEST way to collaborate is to agree on an API and then divide the files to implement between partners, coordinate the edits between files, and then schedule frequent times to work side-by-side in-person to finish the program. Sitting side-by-side to program, with one person typing and the other person contributing ideas and reviewing code as it is typed in, is so effective it has its own name: "pair programming". =20

Start early and talk often with your partner! This is NOT an assignment that you can do last minute. =20

C. Project Organization

=20

The project is divided into two parts, each with multiple tasks= : =20

Part 1 - Due: April 18 - Planning your Project

=20
  • Project Proposal - Specify what specifically you plan to implement. Your project must use each the following at least once:
    • Object Oriented Design
    • Linked Lists
    • Recursion - Note that, as covered in lecture, Linked Lists can be implemented recursively. This will satisfy Recursion and Linked List requirements.
    • =20
  • Presentation of Project Idea: You will be assigned a TA to oversee you project. Usually this will be one of your section TAs, but look out for an e-mail assigning you a different TA becuase we are need to load-balance projects across staff. You are required to schedule a short meeting with this TA, prepare a very brief (30 seconds to 1 minute) oral description of your project, and be prepared to answer questions on your project. This is an opportunity to get feedback on your project from your TA, make sure it will meet the pset requirements, and that it is achievable. Use this time wisely and take notes on the feedback you get!
  • Parner projects: If you are working with a partner, you will each be assigned a TA. Flip a coin to decide which TA you will meet with, and e-mail both TAs to inform them of your decision.
    • =20
  • Description of Project Idea in README: You will write several paragraphs describing your intended project in the README file for Part 1. This read-me will be submitted through the class dashboard.

Part 2 - Due May 1 - Implementation

=20
  • Final project demo: You will demo your completed project to your assigned TA no later than the final day of reading period (May 3rd). Watch for e-mails from your assigned TA about scheduling the demo. Plan on 5 minutes of demo and 10 minutes of questions, discussions, and code review. Note that the deadline for your demo is later than the deadline for submitting your code, but you must demo the version that you submitted.   • Intermediate project meeting: Please e-mail your TA if you would like to schedule an intermediate project meeting before. If they can't run your project or didn't notice a particularly cool aspect of it because you didn't provide thorough enough instructions, then you won't receive credit for it.   • Submission of Project Source Code: You will submit all source code and other files (images, sounds, etc.) necessary to run your project. Your code must compile in Dr. Java and run successfully with the instructions you provide.
  • Independent Project Participation Surveys: Each partner will INDEPENDENTLY submit a brief survey describing what they contributed to the project, their partner's contribution, and whether they encountered any problems with their partner during completion of the assignment.

D. Project Requirements

=20

The project itself is very open ended, with the exception of the following requirements: =20

  • Your final program must use the each of the following in some meaningful way: 1) Object Oriented Design, 2) Linked Lists, and 3) Recursion.
  • Each of these must be used in a non-arbitrary way.
  • Your program must compile and run in Dr. Java.
=20

In terms of project scope, the project should be around 50% the size/complexity of the last two homework assignments if you are doing it alone. If you are doing a pair project, the size/complexity should be roughly triple a single assignment. Use reasonable judgement on this, and ask one of the TAs if you're in doubt.

Grading: Since everyone will be completing different projects, the projects will be graded against a set of generic criteria, such as:

  • Design: Is the program well-designed, with a sensible division into different classes, and good proper use of public vs. private and static vs. non-static variables and methods. Are the APIs for all objects and parts of the program appropriate for their function? Are the APIs clear? Can the program easily be expanded to support additional functionality (which is indicative of good design)?
    • =20
  • Correctness: Does the program successfully implement the intended project? Is the program complete, representing a finished product? Is it robust to different command line arguments and/or user inputs?
    • =20
  • Implementation: Is the code efficient, with minimal computational cost wherever possible? Is the code well-structured, following proper coding conventions? Is proper error checking implemented throughout the program? Is the code well-documented and proper header comments and code comments?
    • =20
  • Creativity/Interestingness: Is the project creative, or does it address an important or interesting problem, or is it especially entertaining, etc.?
    • =20
  • Effort: Does the project show s= ignificant effort? Does it use skills learned throughout CS 112? Is it sufficiently complex for a student about to finish CS 112? =20
  • Documentation: Was the description of the project clear? Were the instructions for running the project clear and correct?
=20

Project Part 1 will be worth 10% of the Project grade; Part 2 will be worth the remaining 90% of the Project grade. The project grade will be factored into your overall homework score and will count as 1.5 psets. This score will not be dropped, even if it is your lowest score. If you do not complete Part 1 before the due date, you cannot receive credit for Part 2.

=20

For this project, we strongly recommend tha= t you focus on just applying everything you've learned throughout CS 112 to complete an interesting project well. Don't worry about trying to meet a particular set of grading criteria. A simple project that is extremely well executed can earn a perfect score. Conversely, a very difficult project that is done poorly will not be scored highly.

E. Project Ideas

=20 You are very welcome (and encouraged) to come up with your own project ideas, but here are a few examples to get you started. Be creative, but remember you only have two weeks! =20

Game Pick a simple game (asteroid, space invaders, snake, checkers, battleship, hangman, minesweeper, etc.) and imp= lement it. Keep it simple, building up your code and the game's functionality as you go. Add your own special twist to the game. This could be a game that uses simple keyboard controls, mouse controls, or a combination both.

=20 =20

Decision Aid Write a program that helps people make a key decision in their life, such as what house to purchase, what car to buy, etc. For example, your program could load data on housing prices, crime rates, school ratings, etc., overlaying them on a map, where the user could turn on and off different layers of the map and input certain criteria to filter the information that is displayed.

=20

Image Editor Write your own image editor (a la PhotoShop) that you could use to load, manipulate, and save images. Add your own brushes, image filters, etc.

=20

PhotoMosaic Write a program that loads an image and reconstructs it as a photomosaic, using a library of source photos. Add some interaction to give the user control over the photomosaic.

=20

Data Visualization Identify an interesting data set on the web (box office revenues, stock prices, annual rainfall, song lyrics, etc.) and visualize it in an interesting way. Your program could load the data, compute statistics on it, and then visualize that data as a graph, colored map, or word cloud. Make the visualization interactive, where the user can interact with it using the mouse, or make it dynamic. Have your program load data dynamically from the internet instead of just from a file. Explore the connection between multiple different data sets, stocks, or songs.

=20

Custom News (Advanced) Grab live news feeds from the internet, filter them against the user's preferred topics, and display a customized page of news. Allow the user to dynamically change their preferred topics.

=20

Game Player (Advanced) Write a program that will play Connect Four or Checkers against you! This should be able to beat at least a novice human opponent half the time (picking moves completely randomly is too simplistic).

=20

Sudoku Solver (Advanced) Write a program that will efficiently solve Sudoku puzzles (at least 9x9 Sudoku. 6x6 is too trivial)

Once you have a complete design in agreement with your partner. Implement the program slowly, testing as you go. Focus first on the core functionality of your program, adding other features once you're certain that the core functionality works well. =20
